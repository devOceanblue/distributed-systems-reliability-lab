# B-0302 — docker-compose: Kafka(3) + Schema Registry + UI + MySQL + Redis

## Goal
모든 실험을 로컬에서 돌릴 수 있도록 infra를 docker-compose로 구성한다.

## Requirements
- Kafka 3 brokers (replication factor 실험 가능)
- Schema Registry 1
- Kafka UI 1
- MySQL 8 (init DB: `lab`, user root/root)
- Redis (standalone)
- 네트워크/포트는 README의 권장 포트 사용

## Implementation Notes
- Kafka는 KRaft 또는 ZK 방식 중 택1. (추천: KRaft로 간소화)
- 토픽 자동 생성은 off 권장. 대신 bootstrap script로 생성.
- 기본 토픽:
  - account.balance.v1 (RF=3, partitions=3)
  - account.balance.retry.5s (RF=3)
  - account.balance.retry.1m (RF=3)
  - account.balance.dlq (RF=3)

## Deliverables
- `docker-compose.yml`
- `infra/kafka/create-topics.sh`
- `infra/mysql/init.sql` (db 생성)
- `infra/redis/redis.conf` (필요시)

## Acceptance Criteria
- `docker compose up -d` 후:
  - Kafka UI 접속 가능
  - Schema Registry health OK
  - MySQL 접속 가능
  - Redis ping OK
- create-topics 스크립트로 토픽이 생성되고 RF=3이 적용됨
