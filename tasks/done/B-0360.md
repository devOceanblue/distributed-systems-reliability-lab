# B-0360 — Frontend Ops Console: request-id idempotency 실험 경로 구현

## Goal
백엔드 실험 프로젝트에 프론트엔드를 연결하고,
프론트 계층에서도 운영 실험에 필요한 멱등성(request-id 재사용)을 강제할 수 있는 실행 경로를 추가한다.

## Scope
1) `frontend/` 모듈 추가
- 무의존성 SPA + Node 프록시 서버
- command/query 서비스 연동
- request-id 전략(STICKY/ALWAYS_NEW/MANUAL) 제공
- `txId`, `Idempotency-Key`, `X-Request-Id` 동시 전달
- retry same request-id / duplicate x2 실험 버튼 제공

2) 자동 검증
- `node:test` 기반 프론트 코어 테스트
- 실험 시나리오 `E-025` run/assert 추가

3) 문서
- 실험 문서: `experiments/E-025-frontend-request-id-idempotency.md`
- 운영 문서: `Runbook.md`, `README.md`, `frontend/README.md`

## Deliverables
- `frontend/*`
- `scripts/scenarios/E-025.sh`
- `scripts/assert/E-025.sh`
- `scripts/verify/B-0360.sh`
- `experiments/E-025-frontend-request-id-idempotency.md`

## Acceptance Criteria
- 프론트에서 동일 request-id 재시도 시 duplicate 결제가 차단됨을 시뮬레이션에서 100% 재현
- 프론트 테스트와 E-025 assert가 자동으로 통과
- Runbook에 실행/검증 절차가 포함됨
